Recipe 13: Creating and Deploying a Simple Infrastructure using Terraform
Objective
Deploy a simple AWS resource using Terraform.
Assignment Task
Create an AWS S3 bucket using Terraform.
Guidelines / Steps
1. Create a Terraform project directory.
2. Define the AWS provider configuration.
3. Define a resource block to create an S3 bucket.
4. Run:
terraform init
5. Generate an execution plan:
terraform plan
6. Apply the configuration:
terraform apply
7. Verify in the AWS Console that the bucket is created.
8. Destroy the resource using:
terraform destroy
Expected Outcome

answer---
1.mkdir terraform-s3
2.cd terraform-s3
3.notepad main.tf

# Configure AWS Provider
provider "aws" {
  region = "ap-south-1"   # Mumbai region
}

# Create S3 Bucket
resource "aws_s3_bucket" "my_bucket" {
  bucket = "rutvik-terraform-bucket-12345"   # Change this (must be unique)

  tags = {
    Name        = "MyTerraformBucket"
    Environment = "Dev"
  }
}
4.terraform init && terraform plan && terraform apply

Recipe 16: Provisioning a Web Server using Terraform
Objective
Deploy a web server on an EC2 instance.
Assignment Task

Create an EC2 instance and configure Apache or Nginx web server.
Guidelines / Steps
1. Create Terraform configuration for EC2.
2. Configure Security Groups to allow:
o SSH (port 22)
o HTTP (port 80)
3. Add user_data script to install a web server.
4. Initialize Terraform.
5. Deploy the infrastructure.
6. Obtain the public IP address of the instance.
7. Access the web server using a browser.
Expected Outcome
A working web server accessible through a public IP.


1.mkdir terraform-webserver
2.cd terraform-webserver
3.notepad main.tf

provider "aws" {
  region = "ap-south-1"
}

resource "aws_security_group" "web_sg" {
  name        = "web-sg"
  description = "Allow SSH and HTTP"

  ingress {
    description = "SSH"
    from_port   = 22
    to_port     = 22
    protocol    = "tcp"
    cidr_blocks = ["0.0.0.0/0"]
  }

  ingress {
    description = "HTTP"
    from_port   = 80
    to_port     = 80
    protocol    = "tcp"
    cidr_blocks = ["0.0.0.0/0"]
  }

  egress {
    from_port   = 0
    to_port     = 0
    protocol    = "-1"
    cidr_blocks = ["0.0.0.0/0"]
  }
}

resource "aws_instance" "web_server" {
  ami           = "ami-0f58b397bc5c1f2e8"   # Amazon Linux 2
  instance_type = "t3.micro"
  key_name      = "my-keypair"   # must exist

  security_groups = [aws_security_group.web_sg.name]

  user_data = <<-EOF
              #!/bin/bash
              yum update -y
              yum install httpd -y
              systemctl start httpd
              systemctl enable httpd
              echo "<h1>Welcome to Terraform Web Server</h1>" > /var/www/html/index.html
              EOF

  tags = {
    Name = "Terraform-WebServer"
  }
}

4.create an =EC2 → Key Pairs → Create Key Pair
5.terraform init && terraform plan && terraform apply
6.aws->ec2->instance->then ip addresh

Recipe 20: Using Terraform to Configure Load Balancers
Objective
Deploy an AWS load balancer.
Assignment Task
Create an Application Load Balancer (ALB) to distribute traffic across EC2 instances.
Guidelines / Steps
1. Create multiple EC2 instances.
2. Define a load balancer resource.
3. Create:
o Target groups
o Listener configuration
4. Attach EC2 instances to the target group.
5. Deploy infrastructure.
6. Test load balancing using browser requests.
Expected Outcome
Traffic should be distributed across multiple instances.

provider "aws" {
    region = "ap-south-1"
}



resource "aws_security_group" "lb_sg" {
    name = "lb-security-group"

    ingress {
        from_port   = 80
        to_port     = 80
        protocol    = "tcp"
        cidr_blocks = ["0.0.0.0/0"]
    }

    ingress {
        from_port   = 22
        to_port     = 22
        protocol    = "tcp"
        cidr_blocks = ["0.0.0.0/0"]
    }

    egress {
        from_port   = 0
        to_port     = 0
        protocol    = "-1"
        cidr_blocks = ["0.0.0.0/0"]
    }
}


data "aws_vpc" "default" {
    default = true
}

data "aws_subnets" "default" {}


resource "aws_instance"  "server1" {
    ami             = "ami-048f4445314bcaa09"
    instance_type   = "t3.micro"
    key_name        = "load-balancer-key"

    security_groups = [aws_security_group.lb_sg.name]

    user_data   =   <<-EOF
                    #!/bin/bash
                    yum update  -y
                    yum install httpd -y 
                    echo "Server 1 Running" > /var/www/html/index.html
                    systemctl start httpd
                    systemctl enable httpd
                    EOF
    
    tags = {
        Name = "Server-1"
    }
}

resource "aws_instance"  "server2" {
    ami             = "ami-048f4445314bcaa09"
    instance_type   = "t3.micro"
    key_name        = "load-balancer-key"

    security_groups = [aws_security_group.lb_sg.name]

    user_data   =   <<-EOF
                    #!/bin/bash
                    yum update  -y
                    yum install httpd -y 
                    echo "Server 2 Running" > /var/www/html/index.html
                    systemctl start httpd
                    systemctl enable httpd
                    EOF
    
    tags = {
        Name = "Server-2"
    }
}

resource "aws_lb" "my_lb" {
    name                = "my-load-balancer"
    internal            = false
    load_balancer_type  = "application"

    security_groups = [aws_security_group.lb_sg.id]
    subnets         = data.aws_subnets.default.ids

    tags = {
        Name = "My_Load_Balancer"
    }
}


resource "aws_lb_target_group" "tg" {
    name        = "my-target-group"
    port        = 80
    protocol    = "HTTP"
    vpc_id      = data.aws_vpc.default.id

    health_check {
        path    = "/"
        port    = "80"
    }

    tags = {
        Name = "Target-Group"
    }
}


resource "aws_lb_target_group_attachment" "server1_attach" {

    target_group_arn    = aws_lb_target_group.tg.arn
    target_id           = aws_instance.server1.id
    port                = 80
}

resource "aws_lb_target_group_attachment" "server2_attach" {

    target_group_arn    = aws_lb_target_group.tg.arn
    target_id           = aws_instance.server2.id
    port                = 80
}
resource "aws_lb_listener" "listener" {

    load_balancer_arn   = aws_lb.my_lb.arn
    port                = 80
    protocol            = "HTTP"

    default_action {
        type                = "forward"
        target_group_arn    = aws_lb_target_group.tg.arn
    }
}

output "load_balancer_dns" {
    value = aws_lb.my_lb.dns_name
}

Recipe 21: Managing Infrastructure Dependencies with Terraform
Objective
Understand resource dependencies in Terraform.
Assignment Task
Create infrastructure with dependent resources.
Guidelines / Steps
1. Create resources such as:

o VPC
o Subnet
o EC2 instance
2. Use implicit or explicit dependencies.
3. Configure Terraform resource relationships.
4. Run:
terraform plan
5. Apply configuration.
6. Verify creation order of resources.
Expected Outcome
Resources should be created in the correct order.

1.mkdir terraform-dependency
2.cd terraform-dependency
3.notepad main.tf

provider "aws" {
  region = "ap-south-1"
}

resource "aws_vpc" "my_vpc" {
  cidr_block = "10.0.0.0/16"

  tags = {
    Name = "MyVPC"
  }
}

resource "aws_subnet" "my_subnet" {
  vpc_id            = aws_vpc.my_vpc.id   # implicit dependency
  cidr_block        = "10.0.1.0/24"
  availability_zone = "ap-south-1a"

  tags = {
    Name = "MySubnet"
  }
}

resource "aws_security_group" "sg" {
  name   = "my-sg"
  vpc_id = aws_vpc.my_vpc.id   # implicit dependency

  ingress {
    from_port   = 22
    to_port     = 22
    protocol    = "tcp"
    cidr_blocks = ["0.0.0.0/0"]
  }
}

resource "aws_instance" "my_ec2" {
  ami           = "ami-0f58b397bc5c1f2e8"
  instance_type = "t3.micro"

  subnet_id              = aws_subnet.my_subnet.id
  vpc_security_group_ids = [aws_security_group.sg.id]

  key_name = "my-keypair"

  tags = {
    Name = "Dependency-EC2"
  }
}
