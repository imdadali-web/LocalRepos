git merge dev

git add conflict.txt
git commit -m "Main branch conflict file"

GitHub Collaboration Workflow
• Create repository on GitHub
• Push local code to remote
• Clone repository to another directory
• Create a feature branch and push it
• Create pull request and review changes
• Merge pull request

STEP 1: Create Repository on GitHub
1.	Open https://github.com
2.	Click New Repository
3.	Enter repository name: github-collaboration
4.	Select Public
5.	Click Create Repository
STEP 2: Create Local Project and Push Code
mkdir github-collaboration
cd github-collaboration
git init
echo "Hello GitHub" > index.txt
git add index.txt
git commit -m "Initial commit"
Add remote repository
git remote add origin https://github.com/username/github-collaboration.git
Check remote
git remote -v
Push code
git branch -M main
git push -u origin main
STEP 3: Clone Repository to Another Directory
cd ..
git clone https://github.com/username/github-collaboration.git
cd github-collaboration
git log
STEP 4: Create Feature Branch and Push
git checkout -b feature-login
echo "Login feature added" > login.txt
git add login.txt
git commit -m "Added login feature"
git push origin feature-login
STEP 5: Create Pull Request
1.	Open GitHub repository
2.	Click Compare & Pull Request
3.	Add Title: Add login feature
4.	Add Description: Added login functionality
5.	Click Create Pull Request
STEP 6: Review Changes
1.	Open Files Changed
2.	Check login.txt
3.	Add comments if needed
4.	Approve changes
STEP 7: Merge Pull Request
Click Merge Pull Request
Click Confirm Merge
Pull Request merged successfully
STEP 8: Update Local Repository
git checkout main
git pull origin main
